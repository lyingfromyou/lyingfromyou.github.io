# Mysql事务

### 事务特性(ACID)

事务具有四个特征：原子性（ Atomicity ）、一致性（ Consistency ）、隔离性（ Isolation ）和持续性（ Durability ）。这四个特性简称为 ACID 特性。



* **原子性(Atomicity)**
  * 一个事务内, 对数据的修改, 要么全部执行, 要么全部不执行

* **一致性(Consistency)**
  * 事务执行后, 没有破坏数据库的一致性(转账-> 我扣500, 另一个人一定会加500)

* **隔离性(Isolation)**
  * 多个事务同时执行, 相互之间没有干扰

* **持久性(Durability)**
  * 事务执行成功后, 数据会永久保存



### 事务隔离级别

* **读未提交(Read Uncommitted)**

  * 所有事务都可以读到其它未提交事务的执行结果, 读未提交的数据, 也被称为脏读. 

* **读已提交(Read Committed)**

  * 事务只能读到其他已经提交的事务, 会产生不可重复读

* **可重复读(Repeatable Read)**

  * Mysql默认的事务隔离级别, 它确保同一事务在多次读取数据的时候, 返回的数据都是一致的, 所以会产生幻读. 

* **序列化(Serializable)**

  * 最高的隔离级别, 通过强制事务排序, 从而解决数据的正确性问题, 通过加锁实现, 性能差.

  

  

  | 事务隔离级别 | 异常 |    异常    | 异常 |
  | :----------: | :--: | :--------: | :--: |
  |   读未提交   | 脏读 | 不可重复读 | 幻读 |
  |   读已提交   |      | 不可重复读 | 幻读 |
  |   可重复读   |      |            | 幻读 |
  |    序列化    |      |            |      |



### 隔离级别所带来的问题(大多数博客没有讲清楚的问题)

举例数据

|  id  | name |
| :--: | :--: |
|  1   | 张三 |



* **脏读**
  * a/b两个事务, a执行 **update** 将id为1的数据改为 **aaa**, 且a没有 **commit**, b在执行 **select** 操作时, 仍然会读到 id为1的名称为 **aaa**, 此时b读到 **aaa** 就是脏读
* **不可重复读**
  * a/b两个事务, a执行 **update** 将id为1的数据改为 **aaa**, 且a没有 **commit**, b在执行 **select** 操作时, 读到id 为1的数据名称仍然是 **张三**, a在执行 **commit** 提交事务后, b再次读取id为1的数据, 发现id为1的数据变成了 **aaa**, 所以b在同一个事务中读取id为1的数据, 两次读取的数据不一致, 所以是不可重复读. 
* **幻读**
  * a/b两个事务, a/b 同时select, 发现表中没有数据, 此时, a 插入一条 id =1, name="张三", 并且commit, 由于b在select的时候, 读不到a对数据的变更, 所以b也插入一条 id = 1的数据, 此时就会报错 **Duplicate entry '1' for key 'PRIMARY'**, 因为在 **可重复读** 的隔离级别中, 会保证同一个事务, 多次读取的数据是相同的, 所以不会读到别的事务已经提交的变更. 



> 最迷惑的就是 **不可重复读** 和 **幻读**,  一个是 -> 在事务内可以读到其他事务已经提交的数据, 一个是由于会保证同一事务内多次读取的数据必须一致, 所以读不到其他事务已经提交的数据.
